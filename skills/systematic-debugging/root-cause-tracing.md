# 根因追踪

## 概述

Bug 通常表现在调用栈深处。你的本能是在错误出现的地方修复，但那只是治标。

**核心原则：** 沿着调用链反向追踪，直到找到最初的触发点，然后在源头修复。

## 追踪流程

### 1. 观察症状
观察错误信息和位置。

### 2. 找到直接原因
哪段代码直接导致了这个错误？

### 3. 问：谁调用了它？
逐层向上问：`A → B → C`，C 出错，B 调了 C，A 调了 B。找到最初的触发点。

### 4. 检查传入了什么值
关注参数值——非法数据通常来自上层。

### 5. 找到最初的触发点
沿着调用链一路向上，直到找到源头。

## 关键原则

- 绝不只在错误出现的地方修复
- 在源头修复
- 如果可能，同时添加纵深防御
