# 坏人测试 — 假设所有输入都是恶意

> 安全领域常用方法：假设用户/调用者/第三方都是恶意的，看系统能否扛住。

## 适用场景

- 安全审计
- 输入校验设计
- API/接口设计
- 边界条件验证

## 执行流程

1. **身份坏人** — 假设调用者是攻击者
   - 注入 SQL/Shell/XSS
   - 绕过权限校验
   - 伪造请求
2. **输入坏人** — 假设数据是恶意的
   - 超长输入、空输入、特殊字符
   - 恶意文件上传
   - 加密/签名篡改
3. **操作坏人** — 假设用户反复恶意操作
   - 重复提交
   - 超频调用
   - 资源耗尽
4. **依赖坏人** — 假设第三方组件被攻破
   - 上游数据污染
   - CDN/库被替换
   - 证书过期

## 核心原则

```
- 不相信任何输入
- 默认拒绝，白名单放行
- 最小权限原则
- 失败安全（fail safe）
```

## 输出

```markdown
## 坏人测试结果

| 攻击面 | 风险等级 | 当前防御 | 改进建议 |
|--------|---------|---------|---------|
| SQL 注入 | 高 | 无参数化查询 | 使用 ORM 或 prepared statement |
| 超频调用 | 中 | 无限流 | 加令牌桶限流 |
```
