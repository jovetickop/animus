# SCAMPER — 从 7 个角度激发改进灵感

> 源自 Bob Eberlee 的 SCAMPER 检核表法。适合对现有功能做改进/优化，而不是从零创建。

## 适用场景

- 现有功能需要迭代优化
- UI/UX 改版
- 流程/架构性能提升

## 7 个检查角度

| 字母 | 关键词 | 核心问题 | 示例 |
|------|--------|---------|------|
| S | **Substitute**（替换） | 能不能换掉某个组件/人/规则？ | JSON → Protocol Buffers |
| C | **Combine**（组合） | 能不能合并两个功能/数据源？ | 用户管理 + 权限管理 |
| A | **Adapt**（适配） | 能不能借鉴其他产品的做法？ | 参考 VS Code 的插件机制 |
| M | **Modify**（修改） | 能不能放大/缩小/改颜色/改比例？ | 减少 API 调用次数 |
| P | **Put to other uses**（他用） | 能不能用在别的地方？ | 导出功能用在报表系统 |
| E | **Eliminate**（消除） | 能不能删掉某个功能/组件？ | 移除无用配置项 |
| R | **Reverse**（反向） | 能不能反过来做？ | 推 → 拉，同步 → 异步 |

## 执行流程

1. 选定要改进的功能/模块
2. 逐条跑 7 个角度
3. 每个角度输出 1-3 个改进点子
4. 汇总后评估：可行性 + 影响范围

## 输出

```markdown
## SCAMPER 结果

### Substitute
- [点子1] 可行性：高/中/低

### Combine
- [点子2] ...

### 最终推荐
1. ...
2. ...
```
