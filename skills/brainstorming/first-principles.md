# 第一性原理 — 拆解到不可再分

> 源自亚里士多德物理学和马斯克工程哲学。适合处理复杂/模糊问题，打破思维惯性。

## 适用场景

- 现有方案成本过高或效果不佳
- 团队陷入「一直这样做的」思维定势
- 需要从零设计新方案

## 核心步骤

1. **拆解到第一性**
   - 把问题分解到「已知为真的基本事实」
   - 区分「物理定律」和「行业惯例」
   - 不断问「为什么」直到不能再分
2. **从基本事实重新构建**
   - 忽略现有实现和约束
   - 从基本组件开始，能构建出什么？
   - 哪些组件的成本在本质上无法降低？
3. **对比现状**
   - 第一性方案 vs 当前方案的成本/复杂度
   - 差距在哪里？是否需要折中？

## 示例

```markdown
问题：组件加载太慢

分解：
- 事实1：数据源在 3 个不同系统 → 能否缓存/合并？
- 事实2：渲染需要 200ms → 能否流式渲染？
- 惯例（非事实）：每次加载必须全量 → 能否增量？

重新构建：只加载用户可见部分 + 后台预取 → 首屏时间降低 80%
```
