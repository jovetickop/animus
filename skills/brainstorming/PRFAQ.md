# PRFAQ — 从用户视角验证产品假设

> 源自 Amazon Working Backwards 方法。适合在规划新功能前验证用户需求和价值主张。

## 适用场景

- 新产品/新功能立项前
- 需要验证用户痛点和解决方案匹配度
- 团队对需求优先级有分歧时

## 执行流程

1. **起草新闻稿**（Press Release）— 模拟功能上线后的新闻稿
   - 标题 + 副标题（一句话价值）
   - 摘要（解决了什么问题）
   - 客户故事（谁在什么场景下受益）
   - 关键指标（如何衡量成功）
2. **模拟客户 FAQ** — 从客户视角提问
   - 负面问题："这意味着我要重新学习吗？"
   - 技术问题："和现有系统怎么集成？"
3. **模拟内部 FAQ** — 从利益相关者视角提问
   - 投入产出比
   - 风险与替代方案
4. **判决** — 继续 / 修改 / 放弃

## 输出

```markdown
## PRFAQ 摘要

### 新闻稿
**标题：** [功能名称] 正式发布
**价值：** [一句话]

### 关键发现
- 用户最关心的：...
- 最大风险：...
- 结论：继续 / 修改 / 放弃
```
