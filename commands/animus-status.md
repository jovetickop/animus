# /animus-status 查看迭代状态

## 功能

显示当前迭代的任务进度总览和依赖关系树，帮助快速了解项目状态和阻塞链。

## 流程

1. **检查运行时目录**：确认 `.claude/animus/` 存在且包含 `features.json`
   - 如果不存在，引导用户先运行 `/animus-setup`
2. **执行状态脚本**：运行 `animus-status.ps1` 调用 Python 状态展示工具
3. **展示输出**：显示带 ANSI 彩色的进度看板和依赖树

## 参数

| 参数 | 说明 |
|------|------|
| `--summary` | 仅显示统计数字摘要（总数/通过/失败/待处理） |
| `--tree` | 仅显示依赖树，隐藏进度头 |
| `--json` | 以 JSON 格式输出原始任务数据 |

不传参数时显示完整看板（进度头 + 依赖树）。

## 输出示例

```
╔══════════════════════════════════════════╗
║  animus 迭代 I3                  2026-07-04 ║
║  ──────────────────────────────────────  ║
║  进度: ████████████░░░░  73% (11/15)     ║
║  ✅ 8  ❌ 2  🟡 1  ⏳ 4                   ║
╚══════════════════════════════════════════╝

依赖树:
T001 ✅ 任务名 (passed)
T002 ✅ 任务名 (passed)
│
└── T003 🟡 任务名 (in_progress)
    │
    ├── T004 ⏳ 任务名 (pending)
    │   │
    │   └── T005 ⏳ 任务名 (pending) [dep: T003]
    │
    └── T006 ⏳ 任务名 (pending)

阻塞链摘要:
  T006 ← T005 ← T003,T004  (3 层)
```

每个任务在树中只出现一次（spanning tree），依赖关系通过缩进和连线展示。

## 颜色约定

| 颜色 | 含义 |
|------|------|
| 绿色 | 已通过 |
| 红色 | 失败 |
| 黄色 | 进行中 |
| 灰色 | 待处理 |
