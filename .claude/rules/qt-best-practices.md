# Qt 最佳实践

## 对象所有权

- 堆上创建的 `QObject` 尽量明确父对象或生命周期归属。
- UI 对象图优先使用 Qt 的父子关系管理，不要到处分散手动释放。
- 避免保存生命周期不清晰的裸指针。

## 信号与槽

- 优先使用函数指针形式的新式连接语法。
- 槽函数保持轻量，耗时逻辑移出 UI 线程。
- 跨线程通信时，明确使用合适的连接方式。

## UI 与资源

- `.ui`、源码和资源文件改动必须保持同步。
- MOC、UIC、RCC 失败视为当前任务失败，不能跳过。
- 对长耗时操作提供进度或状态反馈。

## 工程约束

- 保持现有架构选择稳定：Qt Widgets 项目继续使用 Widgets，QML 项目继续使用 QML，除非任务明确要求调整。
- 避免 UI 控件层直接耦合复杂业务逻辑。
