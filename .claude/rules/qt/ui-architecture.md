# UI 架构规范

## 布局优先

- 主结构必须优先使用 Qt Layout。
- 不要依赖绝对定位组织主界面。
- 主要内容区默认允许横向和纵向拉伸，除非设计有明确限制。

## 对话框与面板边界

- 对话框用于短流程、明确决策或少量数据输入。
- 长驻工作流优先放在主窗口或侧边面板中。
- 危险操作与主操作要有明显区分。

## 按钮与反馈

- 对话框中主确认按钮放在右侧。
- 次要按钮要可发现，但不能比主按钮更抢眼。
- 对明显耗时的动作提供状态栏、行内提示、进度条或日志反馈。

## 尺寸与文案

- 除非控件有真实最小需求，否则避免写死尺寸。
- 文本输入、表格、日志区等控件要显式设置合适的 `sizePolicy`。
- 标签、按钮和提示文案保持中性、明确、无歧义。
