description: 围绕当前任务结果执行构建、测试、Qt 和 UI 四类验收检查
---

请把当前改动作为“当前任务的验收门”来审查。

## 审查范围

重点检查：

- 构建是否正确
- 测试覆盖和测试命令是否合理
- Qt 对象所有权与信号槽使用是否正确
- UI 布局、拉伸行为和交互反馈是否合理
- harness 状态是否一致

## 审查清单

1. 先确认 `.claude/harness/features.json` 中的当前任务。
2. 检查代码、UI、测试是否满足该任务的验收标准。
3. 验证构建与测试命令是否可执行、是否足够具体。
4. 检查 Qt 风险：
   - `QObject` 所有权
   - UI 线程是否被耗时逻辑阻塞
   - 信号槽连接方式
   - MOC、UIC、RCC 是否仍然正确
5. 检查 UI 风险：
   - 是否布局优先
   - 是否存在不必要的固定尺寸
   - 按钮位置与状态反馈是否合理
   - 长耗时操作是否提供状态栏、进度条或日志反馈
6. 确认 `.claude/harness/claude-progress.txt` 已记录最新结果。
7. 确认 `docs/reports/<任务编号>-任务描述.md` 已生成并包含“功能描述 + 最新验证结果”。

## 输出格式

对每个问题给出：

- 严重程度：`high`、`medium` 或 `low`
- 文件与行号（如果能确定）
- 问题说明
- 影响
- 修复建议

如果没有发现问题，也要明确说明，并补充剩余测试缺口。
