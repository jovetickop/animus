# 优化任务 ⑧：头脑风暴 / 产品发现

> 对应路线图：Phase 3 — 深度建设
> 解决：零 ideation 能力，用户只能自己想清楚需求

---

## 一、更改原因

### 1.1 当前问题

- 用户说"我想做 X 但不知道从哪开始"时，系统无法帮助理清需求
- 所有需求输入都是用户单向的，系统不会主动提问帮助澄清
- 缺乏结构化的 ideation 技法

### 1.2 解决后的效果

- 集成到 `/animus-dev --full` 的 7 问之前
- 6 个头脑风暴技法覆盖不同场景
- 脑暴结果自动写入 features.json 的 spec 字段

---

## 二、更改方案

### 2.1 定位

集成到 `/animus-dev --full` 的 7 问 Grilling 之前。当用户说"不确定""不知道"时自动启用，**触发前询问用户是否启动脑暴**。

### 2.2 6 个技法

| # | 技法 | 适用场景 | 核心问题 | 输出 |
|---|------|---------|---------|------|
| 1 | **PRFAQ** | "不知道做什么" | 假设功能已上线，写一篇发布公告，描述用户获得了什么价值 | 假想发布公告 + 用户价值列表 |
| 2 | **第一性原理** | "方案太复杂" | 把问题拆到不可再拆的基本元素，从基础重新构建方案 | 基本元素列表 + 新方案草图 |
| 3 | **SCAMPER** | "优化已有功能" | 七问检查：替代？结合？改造？换用？挪作他用？消除？反向？ | 七问检查表 + 优化建议 |
| 4 | **逆向思维** | "卡住了" | 如果解决不了这个问题，我们能不能换个目标？ | 替代目标 + 可行性对比 |
| 5 | **坏人测试** | "不确定什么是好" | 怎么把这个功能搞坏？从失败反推成功标准 | 失败场景列表 + 成功标准 |
| 6 | **竞品评测** | "同质化竞争" | 假装在写竞品的产品评测，找到自己的差异点 | 竞品分析 + 差异化清单 |

### 2.3 集成方式

在 `/animus-dev --full` 流程中：

```
用户输入意图
  → AI 检测是否需要脑暴（用户表示不确定/不清楚/没想好）
  → 询问用户是否启动脑暴
  → 如果同意：
      → AI 自动选择最合适的技法（或用户指定）
      → 执行脑暴技法
      → 结果写入 features.json spec 字段（why / capabilities / constraints / non_goals / success）
  → 如果不同意或不需要：
      → 直接进入标准 7 问 Grilling
```

### 2.4 技法选择逻辑

| 用户说... | 自动选用的技法 |
|-----------|---------------|
| "我想做 X 但不知道从哪开始" | PRFAQ |
| "这个功能有没有更简单的方式" | 第一性原理 |
| "这个功能还能怎么优化" | SCAMPER |
| "搞不定，换个思路" | 逆向思维 |
| "不知道什么样的方案算好" | 坏人测试 |
| "别人也做了，没区别" | 竞品评测 |

### 2.5 输出格式

脑暴结果直接写入 features.json 的 spec：

```json
{
  "T004": {
    "spec": {
      "why": "来自 PRFAQ：客户在飞机上无法访问网络，但仍需要查看报告",
      "capabilities": ["导出 PDF", "离线查看", "选择导出范围"],
      "constraints": ["离线可用", "导出文件不超过 50MB"],
      "non_goals": ["不实现在线编辑", "不支持批量导出"],
      "success": "用户在无网络环境下 3 步内完成 PDF 导出并打开"
    }
  }
}
```

### 2.6 改动文件

| 文件 | 改动 |
|------|------|
| 新建 `skills/brainstorming/` 目录 | 6 个技法实现 |
| 修改 `commands/animus-dev.md` | full-path 7 问前集成脑暴触发逻辑 |

## 三、架构影响评估

| 维度 | 评估 |
|------|------|
| 性能 | 脑暴仅在全量路径且用户确认后启用，不影响日常使用 |
| 兼容性 | features.json spec 字段兼容新旧两种来源（脑暴 / Grilling） |
| 降级 | 脑暴失败/跳过时回退到标准 7 问 Grilling，不阻塞 dev 流程 |

## 四、验证方法

1. 说"我想做个导出功能但不知道从哪开始" → 触发 PRFAQ 脑暴
2. 说"这个功能太复杂了" → 触发第一性原理
3. 脑暴完成后 → 确认 features.json spec 已填充
4. 拒绝脑暴 → 确认直接进入标准 7 问 Grilling
5. 脑暴过程中中断 → 确认回退到 Grilling 不报错
